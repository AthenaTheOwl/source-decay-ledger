# Design — 0002-design

## Architecture

Five blocks, four files of Python source:

```
                ┌──────────────────────────────────┐
                │  data/sources.yaml               │  human-edited registry
                └────────────┬─────────────────────┘
                             │ load + validate
                             ▼
                ┌──────────────────────────────────┐
                │  registry.py                     │  pydantic models + loader
                │  - Source                        │
                │  - load_registry(path) -> [..]   │
                │  - validate_registry(path) -> ok │
                └────────────┬─────────────────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
        ▼                    ▼                    ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│  ledger.py      │ │  score.py       │ │  memo.py        │
│  - append(row)  │ │  - score_week() │ │  - write_memo() │
│  - read_week()  │ │  - read_scores()│ │  - diff_vs_prior│
│  - check_append │ │                 │ │                 │
│    _only()      │ │                 │ │                 │
└─────────────────┘ └─────────────────┘ └─────────────────┘
                             │
                             ▼
                ┌──────────────────────────────────┐
                │  cli.py + __main__.py            │  argparse: validate /
                │                                   │  append / append-only-
                │                                   │  check / score / memo /
                │                                   │  --dry-run (full loop)
                └──────────────────────────────────┘
```

Each block is a single python module under `src/source_decay_ledger/`. The CLI composes them; no module imports the CLI.

## Data schemas (Pydantic models)

```python
class Source(BaseModel):
    slug: str = Field(pattern=r"^[a-z0-9][a-z0-9-]{2,40}$")
    name: str
    category: str
    fetch_kind: Literal["rss", "atom", "html", "manual"]
    fetch_target: str
    added_on: date
    verdict: Literal["keep", "probation", "drop"]
    aliases: list[str] = []

class LedgerRow(BaseModel):
    item_slug: str  # autogenerated from evidence_url hash
    published_on: date
    source_slug: str
    evidence_url: str
    appended_at: datetime  # UTC

class ScoreRow(BaseModel):
    source_slug: str
    count_90d: int
    count_30d: int
    last_seen_at: datetime | None
```

## Append-only invariant

`data/ledger/.manifest.jsonl` carries one row per committed ledger row: `{week, item_slug, sha256}`. On `append`, the new row is computed-hashed and appended to BOTH the week's ledger file AND the manifest. On `append-only-check`, every existing manifest row's hash is recomputed from the live ledger file and compared. Any mismatch fails the check.

The manifest is committed; tampering it is detectable via git diff.

## Failure modes per block

| Block | Failure mode | Behavior |
|---|---|---|
| registry.py | sources.yaml missing fields | exit 2 with offending entry slug |
| registry.py | invalid verdict value | exit 2 with line number |
| registry.py | duplicate slug | exit 2 |
| ledger.py | week file unwritable | raise IOError; CLI exits 3 |
| ledger.py | append-only check fails | exit 4 with first mismatched row |
| score.py | week argument has no ledger file | warn + write empty score file |
| memo.py | no prior week file | "no prior week" instead of diff |
| cli.py | invalid week format | exit 2 |
| cli.py | --dry-run when output already exists | print "would overwrite", exit 0 |
